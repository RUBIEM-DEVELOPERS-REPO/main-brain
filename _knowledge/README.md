# _knowledge/

Reusable patterns, gotchas, recipes — practical know-how that's project-agnostic.

One file per topic (e.g. `git-workflows.md`, `obsidian-plugins.md`, `windows-shell-gotchas.md`). Append as you learn.

This is the boring-but-useful counterpart to `2-Permanent/`. Knowledge lives here when it's "how to do X" rather than "an idea worth thinking about."
