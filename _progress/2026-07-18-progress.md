# Progress — 2026-07-18: NeuroWorks always-on

Goal (user request): live chat + agents always available — auto-start at logon, auto-restart on crash.

## What shipped
- **Watchdog**: `tools/always-on/neuroworks-always-on.ps1` (in the neuroworks repo, Documents/GitHub/clawbot). Restores the D: vault subst first, then supervises the server in production single-port mode (`SERVE_WEB=1`, `NEUROWORKS_PORT=7470`) — one process serves API + built web UI at the same URL the dev stack used. Restart-on-exit with 10s backoff; idles politely (30s re-check) if anything else already holds 7470, so `pnpm dev` never fights it. Singleton via PID lock file. Logs to `server/.neuroworks/always-on.log` (2MB cap).
- **Startup shim**: `neuroworks-always-on.vbs` in the user Startup folder — launches the watchdog hidden at every logon. Ollama already auto-starts from the same folder.
- **Stop script**: `tools/always-on/stop-always-on.ps1` — kills watchdog + supervised server tree; refuses to kill a port owner whose command line isn't this repo's server.

## Verified live
1. Prod mode boot-tested on spare port 7999 first (health OK, vault OK, web served) before touching anything.
2. Dev stack (Vite 7470 + dev API 7471 + worker 7473) stopped at 0 in-flight jobs; watchdog took over 7470 within 30s — health, web UI (HTTP 200), and LLM status all green.
3. Crash test: killed the server process — watchdog logged the exit and had it healthy again ~35s later.

## Gotcha (cost one failed launch)
Windows PowerShell 5.1 reads BOM-less UTF-8 scripts as ANSI — em-dashes in the .ps1 mangled to `â€”` and broke a string terminator, so the watchdog silently failed to parse. Fix: keep .ps1 files pure ASCII (or save UTF-8 with BOM).

## Notes
- Server now runs production single-port; the reflection scheduler, workers, and chat all live inside the supervised process.
- To develop again: run `stop-always-on.ps1`, then `pnpm dev` as usual. Watchdog resumes ownership next logon (or re-run the .vbs).
