# NeuroWorks progress — 2026-06-30

Deployment countdown: ~3 days out. Focus: making NeuroWorks an orchestration
layer other systems can dispatch agents into, plus the ADRS data pipeline.

## 1. ADRS = the main data pipeline (shipped, live-verified)
Implemented the AI Data Readiness System as NeuroWorks' main data pipeline.

- **Engine** (`server/src/lib/adrs.ts`): Normalization → Cryptographic Hashing
  (SHA-256 per record + Merkle batch root) → Confidence Scoring → HITL gate →
  Entity Resolution (golden record) → Publish.
- **Outputs** per dataset, written to the vault under `_datasets/<id>/`:
  ML-ready CSV, knowledge-graph JSONL, RAG chunks, and a knowledge-pack card.
- **Agents learn from published datasets**: artifacts land in the vault → the
  vault index picks them up → agents retrieve them. Published datasets also
  surface on the Knowledge Packs page as dataset packs.
- New agent primitives `data.publish` / `data.list_datasets`, a `/api/datasets`
  route, and a `/data-pipeline` web page.
- Verified live: 4 raw records → 3 golden (duplicate id merged, null backfilled,
  whitespace trimmed, types coerced), 1 record HITL-flagged, ~83% avg
  confidence, all four artifacts written.

## 2. External agent dispatch (foundation, live-verified)
NeuroWorks can now be dispatched into from other systems as an orchestration layer.

- **Machine identity** (`server/src/lib/api-keys.ts`): API keys (`nw_…`), stored
  only as SHA-256 hashes, scoped (`dispatch:write` / `dispatch:read`).
- **Async dispatch** (`/api/v1/dispatch`): POST a task → get a jobId → poll or
  receive an HMAC-signed webhook callback. Idempotency-Key support; per-key
  tenancy (a key reads only its own jobs); webhook SSRF guard.
- Verified: unauthenticated → 401, dispatched with no browser Origin (machine
  surface exempt from the origin guard), idempotent repeat returned the same
  jobId, and a different key got 404 on another key's job.
