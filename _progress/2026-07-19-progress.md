# Progress — 2026-07-19: four fixes, all live-verified

## 1. Cross-user chat leak FIXED
- Client cache was global (`neuroworks.chat` localStorage keys) — every account on a shared browser saw the previous user's conversation. Now all three chat keys are namespaced per signed-in user id (`neuroworks.chat.<userId>`), legacy shared keys purged on load. `lib/api.ts` tracks the user id at login / session refresh / logout.
- Server got an authoritative per-user history store (`lib/chat-history.ts`, `.neuroworks/chat-history/<userId>.json`): every chat exchange records under the calling token's user via a res.json intercept in `routes/chat.ts`. Machine callers not recorded. `GET /api/chat/history` serves ONLY the caller's own entries (401 token-less, verified) and joins finished job answers from the job store.

## 2. My Work — "My prompts & results" section
- New section on the My Work page: every prompt the signed-in user asked in chat with the output it produced (expandable rows, persona, job status badge, deep-link to /results/<jobId>). Polls every 15s. Built into web/dist.

## 3. Security watchdog (cyber-attack detection)
- `lib/security-watch.ts` mounted in front of every route: scanner-probe paths (/wp-admin, /.env, /.git…), path traversal, URL SQLi/XSS signatures, auth-failure bursts (401/403 ×10 in 5m), request floods. Per-IP decaying score → auto-block 403 for 30m past threshold. Loopback logged but NEVER blocked (workers/peers legitimately hammer localhost). Events → `.neuroworks/security-events.json` + audit log; `GET /api/security/events`, `POST /api/security/unblock` (admin layer). Live-verified: probes on /wp-admin and /.env logged with correct severity, loopback unblocked.

## 4. Org-knowledge grounding — "I lack details about Aiia's teams" refusal KILLED
Four stacked causes, each found live:
1. No agent-roster primitive at all → added `personas.list` (id, role, JD, responsibilities, work mode; systemPromptOverride redacted).
2. Small triage model classified org questions as DIRECT world-knowledge → confident refusal. Added ORG_STRUCTURE_RE veto in `looksLikeDirectAnswer` + `llmTriage`.
3. LLM planner (gemini) timed out → generic fallback cascade routed to research.deep (web) for a private-company question. Replaced with a DETERMINISTIC heuristic plan: users.list → personas.list → company.department_data → synth. Plus an orgContext hint in plan() for org tasks arriving by other routes.
4. Synthesis truncation: reasoning models (nemotron free tier, gemini-3.5-flash) burned the 512–1024 max_tokens on chain-of-thought → 69–97-char answers of leaked "Wait, is…" reasoning. openrouter.ts now gives reasoning-family models 4× budget (min 4096) and strips inline/unclosed <think> blocks.

Final verification: "Create three job descriptions fitting Aiia's organizational structure with agents incorporated" → 2.6KB deliverable naming real staff (Patience, Anesu), real agents (Liam, Ada, Pax, Bram), reporting lines, human-agent collaboration split, with [1][2] citations to directory/roster evidence. "What departments exist in our organization?" → the 5 real departments from the users directory, honest note that company.department_data is empty.
