# 2026-07-01 — Production hardening (2 days to deployment)

Focused push to get NeuroWorks production-ready + two live-issue fixes.

## Production hardening

- **Optimized static web build (single-port prod).** The container was running
  the Vite **dev server** in production. Now the SPA is built (`vite build`) and
  the API serves the minified, fingerprinted assets itself on one port
  (SERVE_WEB=1, bind 0.0.0.0). SPA history-fallback + long-cache on assets,
  no-cache on index.html, JSON 404 for unknown `/api/*`. Dockerfile + compose
  rewritten to single-port with a `HEALTHCHECK`. Origin-guard updated: static
  GETs pass, same-origin API calls allowed, `CLAWBOT_ALLOWED_HOSTS` for the prod
  host (DNS-rebind defense intact). Live-verified end-to-end on a throwaway port.
- **Runtime resilience.** Graceful shutdown now `server.close()`s to stop new
  connections + a hard-exit safety net (`NEUROWORKS_SHUTDOWN_GRACE_MS`, def 12s)
  so a wedged teardown can't hang the container. (Process guards + job-abort +
  vault-flush were already there.)
- **Config fail-fast (`validateConfig`).** In production/strict mode the server
  refuses to boot on a fatal misconfig (bad port, SERVE_WEB with no build,
  missing required env) with a clear message instead of failing deep in a
  request. Verified: invalid port → clean exit(1).
- **Dispatch API hardening.** Per-key sliding-window rate limit (429 +
  Retry-After, `NW_DISPATCH_RATE_PER_MIN` def 120); 256KB body cap on machine
  endpoints (`/api/v1/`, `/api/public/`) rejected before the 25MB parser buffers
  (413); tighter webhook SSRF — checks ALL resolved addresses, `redirect:manual`
  (no 3xx-to-internal bypass), IPv4-mapped-IPv6 + CGNAT coverage. All verified
  (413, 429, 13-case SSRF predicate).
- **Secrets rotation runbook** at `clawbot/docs/DEPLOYMENT-SECRETS.md`. Both
  repos scanned clean (no committed secrets; `.env` gitignored). Rotate before
  go-live: GitHub PAT, OpenRouter, OpenAI `sk-`, Slack `xoxb-`, Stripe.
