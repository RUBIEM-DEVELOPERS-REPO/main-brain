## Critique of Designs Against the User’s Job‑to‑Be‑Done

### 1. Identify the Core JTBD  
- Determine the specific outcome the user seeks (e.g., “complete a task faster,” “reduce error,” “gain confidence”).  
- Verify that every design element supports this outcome; any feature that does not directly contribute to the JTBD should be reconsidered or removed.  
- This focus ensures the product fulfills a real need rather than merely pleasing aesthetic preferences. [1]

### 2. Measure Outcome‑Oriented Success  
- Use quantitative metrics such as task‑completion time, error rate, or Net‑Promoter Score to assess whether the design delivers the desired result.  
- Evaluate designs against these metrics to confirm that the user’s goal is achieved efficiently and reliably. [1]

### 3. Evaluate Functional Fit  
- Inspect layout, interaction flows, and visual cues for alignment with the JTBD.  
- Remove or redesign any element that introduces friction without adding value, as extraneous features dilute focus and can hinder usability. [1]

### 4. Assess Emotional & Social Dimensions  
- Consider how the design makes the user feel—confidence, control, status—and whether it reinforces the user’s self‑image or social expectations.  
- Emotional resonance can enhance adoption, but it must not override functional effectiveness. [1]

### 5. Iterate Based on Feedback  
- Gather user feedback specifically tied to the JTBD.  
- Refine the design to close gaps between intended outcomes and actual user experience, ensuring continuous alignment with the core job. [1]

**Conclusion**  
A design that prioritizes the user’s job‑to‑be‑done, backed by outcome metrics and functional alignment, will deliver real value and sustain user satisfaction. Personal taste should remain a secondary consideration. [1]
