## Why a Data Analyst Should Form a Hypothesis Before Pulling Data  

| Step | Action | Why It Matters |
|------|--------|----------------|
| 1 | Define a clear research question | Gives direction and prevents endless data fishing |
| 2 | State a null (H₀) and an alternative (H₁) hypothesis | Provides a criterion for statistical testing |
| 3 | Identify required variables and data sources | Limits the data pull to what is needed, saving time and resources |
| 4 | Plan the analysis (method, sample size, power) | Guides how much data to collect and how it should be prepared |
| 5 | Pull only the necessary data | Reduces noise, speeds processing, and respects privacy |
| 6 | Clean and explore the data | Ensures reliability before testing |
| 7 | Test the hypothesis | Delivers an objective answer to the question |
| 8 | Interpret and communicate results | Turns findings into actionable insights |  

**Core Benefits**  
1. **Focus & Efficiency** – Targeted data pulls avoid unnecessary work.  
2. **Statistical Rigor** – Pre‑specified hypotheses prevent p‑hacking.  
3. **Resource Management** – Saves compute, storage, and compliance effort.  
4. **Clear Communication** – Provides a logical chain for stakeholders.  

**Quick Checklist**  
- Write a concise research question.  
- Formulate H₀ and H₁.  
- List needed variables and sources.  
- Choose analysis method and calculate required sample size.  
- Extract only the necessary data.  
- Clean the data (remove outliers, correct errors).  
- Run the test and report results.
