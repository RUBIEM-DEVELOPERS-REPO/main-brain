## Sketch‑Flow Guidelines to Minimise Cognitive Load and Decision Points

### 1. Limit Choices per Step  
- **Progressive disclosure**: reveal only the next essential action; hide optional features behind a “More” or secondary screen.  
- **Primary‑secondary grouping**: place the main action in a prominent button, with secondary actions in a less‑prominent area (e.g., bottom‑sheet or overflow menu).  

### 2. Establish a Clear Visual Hierarchy  
- Use size, colour, and placement to signal the primary path.  
- Keep the number of visual elements per screen to a handful (3‑5) to avoid overload.  

### 3. Leverage Familiar Patterns  
- Adopt well‑known UI conventions (e.g., hamburger menu for navigation, thumb‑reachable tap targets).  
- Re‑use the same interaction pattern for similar tasks throughout the flow.  

These principles keep the experience simple, focused on the core need, and reduce the mental effort required to parse many options or unfamiliar interactions.
