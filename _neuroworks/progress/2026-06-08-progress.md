## Shipped this cycle

### 1. Company-system Connectors (outbound)
- Registry of external HTTP APIs agents can call: base URL + auth scheme (bearer / api-key / basic / custom-header / none) + a self-describing **endpoint manifest**. Secrets encrypted at rest (AES-256-GCM, shared secret-box).
- **Read-only by default** (writes opt-in per connector) + an **SSRF guard** (a call can't escape the connector's base origin).
- Primitives `connector.list` / `connector.describe` / `connector.call`; `Connectors` UI page with a built-in "Try" tester.
- Live-verified: add → test → authenticated GET → cross-origin path rejected → write blocked.

### 2. AIIA finance integration

### 7. Hermes as primary executor (hybrid)
- Runtime toggle `POST /api/executor` (persisted, **no restart**) makes the Hermes agent the primary doer, with the **persona/lane/governance** framing injected.
- **MCP bridge**: `/api/primitives` (16-tool allowlist — vault, connectors → AIIA, integrations, web reads, payment status; money-moving + writes excluded) + a stdio MCP server (`server/mcp/clawbot-mcp.mjs`). Registered with Hermes (`hermes mcp add`). **Verified Hermes reading the AIIA manifest over MCP.**
- **Hermes → clawbot offload**: anything Hermes can't do — errors, "no final response", thin answer, OR a failed `quality.check` — falls through to clawbot's full pipeline. Both paths verified.

## Benchmark findings
- **Hermes's default model (`claude-opus-4.6`) is broken** on its key — every run returned "no final response" (root cause of 0 logged Hermes successes). Pinned a working free model.
- Same-model head-to-head (5 short Q&A): Hermes 5/5 vs clawbot 4/5, faster (27s vs 110s).
- Novel stress harness (12 persona/media/structured probes) with Hermes primary: **67% vs clawbot's ~92%** — Hermes regresses on grounded / media / structured tasks. → Use clawbot for grounded/structured work, Hermes for fast general Q&A; the toggle + offload make this automatic.
