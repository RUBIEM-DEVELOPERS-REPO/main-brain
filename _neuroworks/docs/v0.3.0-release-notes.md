# NeuroWorks v0.3.0 — Release notes

> NeuroWorks becomes an orchestration layer other systems dispatch agents into,
> and ships a data pipeline whose published datasets agents learn from. Plus a
> one-command install for any PC.

## Headline features

### ADRS — the main data pipeline
The AI Data Readiness System is now NeuroWorks' main data pipeline. Raw inputs
flow through Normalization → Cryptographic Hashing (SHA-256 + Merkle batch
root) → Confidence Scoring → HITL gate → Entity Resolution (golden record) →
Publish. Each dataset publishes four machine-ready artifacts into the vault
under `_datasets/<id>/`: ML-ready CSV, knowledge-graph JSONL, RAG chunks, and a
knowledge-pack card.

Published datasets are indexed automatically, so **agents learn from them** via
retrieval, and they surface on the Knowledge Packs page. Agents can run the
pipeline themselves.

- **Page:** `/data-pipeline`
- **API:** `GET/POST /api/datasets`, `GET /api/datasets/:id/output/:kind`
- **Primitives:** `data.publish`, `data.list_datasets`

### Omnisignal — multi-source data acquisition
A base research system and the acquisition front-end of ADRS. Gathers raw
signal from web search, web pages, connected databases, local files, and the
vault; normalizes each into a provenance-tagged record stream; and feeds it
into ADRS in one call. Modelled on the omnis-signal "Scraper Hub" (Source →
scrape → normalized Record → publish) but NeuroWorks-native.

- **Page:** Omnisignal panel on `/data-pipeline`
- **API:** `POST /api/omnisignal/acquire`, `POST /api/omnisignal/publish`,
  `GET/POST/DELETE /api/omnisignal/sources`, `GET /api/omnisignal/kinds`
- **Primitives:** `omnisignal.acquire`, `omnisignal.publish`

### External agent dispatch
NeuroWorks can be dispatched into from other systems as an orchestration layer.

- **Machine identity:** API keys (`nw_…`), stored as SHA-256 hashes, scoped
  (`dispatch:write` / `dispatch:read`).
- **Async dispatch:** `POST /api/v1/dispatch` → `jobId`; poll or receive an
  HMAC-signed webhook. `Idempotency-Key` support; per-key tenancy; webhook
  SSRF guard.
- **API:** `POST /api/v1/dispatch`, `GET /api/v1/dispatch/:jobId`,
  `POST|GET|DELETE /api/dispatch-keys`.

### Models — pull menu + bring-your-own APIs
A `/models` page to pull/remove local Ollama models from a curated catalog
(streamed progress), set the default, and plug in cloud model APIs you already
use (OpenAI, OpenRouter, Groq, Together, or any OpenAI-compatible endpoint).
Provider keys are encrypted at rest and applied to the router with no restart.

- **Page:** `/models`
- **API:** `/api/models/catalog`, `/api/models/pull` (SSE), `/api/models/installed/:name`,
  `/api/models/providers*`

### Download & run on any PC
One-command install with a free local core — no personal keys to start.

- `tools/install.ps1` (Windows), `tools/install.sh` (macOS/Linux)
- [QUICKSTART.md](../../QUICKSTART.md)

### UI refresh
The web console was re-tokenized to the midnight-blueprint design system;
remaining off-brand pages were normalized for visual consistency.
