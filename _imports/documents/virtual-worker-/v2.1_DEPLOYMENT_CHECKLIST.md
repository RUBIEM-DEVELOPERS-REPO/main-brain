### Phase 2: Frontend Updates (15-30 minutes)

#### 2.1 Update Response Handling

**Old Code (v2.0):**
```javascript
const res = await fetch('http://localhost:3001/main-command', {...});
const data = await res.json();
const result = data.data;  // ❌ OLD
```

**New Code (v2.1):**
```javascript
const res = await fetch('http://localhost:3001/main-command', {...});
const data = await res.json();
const result = data.execution;  // ✅ NEW

// Also available:
console.log(data.reasoning);    // Department, intents, confidence
console.log(data.timestamp);    // Response timestamp
```

#### 2.2 Update Result Display Component

**Old Component:**
```jsx
<div>{data.data?.message}</div>
```

**New Component:**
```jsx
{data.execution && (
  <div className="execution-result">
    <div className="header">
      <h2>{data.execution.title}</h2>
      <span className={`status ${data.execution.status}`}>
        {data.execution.status.toUpperCase()}
      </span>
    </div>

    <p className="section">{data.execution.section}</p>

    {/* Display based on department-specific type */}
    {data.execution.results?.type === 'financial' && (
      <FinancialView data={data.execution.results.data} />
    )}
    {data.execution.results?.type === 'hr' && (
      <HRView data={data.execution.results.data} />
    )}
    {data.execution.results?.type === 'sales' && (
      <SalesView data={data.execution.results.data} />
    )}

    {/* Action buttons */}
    <div className="actions">
      {data.execution.results?.actions?.map(action => (
        <button key={action} onClick={() => handleAction(action)}>
          {action}
        </button>
      ))}
    </div>

    {/* Error details if failed */}
    {data.execution.status === 'error' && (
      <div className="error-details">
        <p>{data.execution.message}</p>
        {data.execution.details && (
          <code>{data.execution.details.errorCode}</code>
        )}
      </div>
    )}
  </div>
)}
```

## 📊 Database Monitoring Queries

### Query 1: Check Recent Executions
```sql
SELECT 
  id, user_id, department, status, confidence, created_at
FROM executions
ORDER BY created_at DESC
LIMIT 5;
```

### Query 2: Success Rate
```sql
SELECT 
  COUNT(*) as total,
  SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) as successful,
  ROUND(100.0 * SUM(CASE WHEN status = 'success' THEN 1 ELSE 0 END) / COUNT(*), 2) as success_rate
FROM executions
WHERE created_at > NOW() - INTERVAL '24 hours';
```

### Query 3: Department Distribution
```sql
SELECT 
  department, 
  COUNT(*) as count,
  AVG(confidence) as avg_confidence
FROM executions
GROUP BY department
ORDER BY count DESC;
```

## 🚨 Troubleshooting

### Issue: "Could not find the table 'public.executions'"

**Solution:**
1. SQL schema hasn't been executed yet
2. Go to Supabase SQL Editor
3. Run SUPABASE_SCHEMA_SETUP.sql
4. Restart backend

### Issue: "Connection refused on localhost:3001"

**Solution:**
```bash
# Check if backend is running
lsof -i :3001

# If not running:
npm start

# Check if port is in use:
# macOS/Linux: lsof -i :3001
# Windows: netstat -ano | findstr :3001
```

### Issue: Database shows "connected: false"

**Solution:**
1. Verify Supabase credentials in .env
2. Verify tables are created in Supabase
3. Check internet connection
4. Restart backend: `npm start`

### Issue: n8n webhook returns 404

**Solution:**
1. Verify n8n is running
2. Check webhook URL in .env
3. Verify webhook is activated in n8n
4. Check n8n logs for errors

### Issue: Frontend receives old format

**Solution:**
1. Hard refresh browser (Ctrl+Shift+R)
2. Clear browser cache
3. Verify backend is running v2.1
4. Check `/health` shows "response-formatting" in features

For production, consider:
```javascript
// In server.js, add connection pooling:

  db: {
    schema: 'public',
  },
  global: {
    headers: { 'Content-Type': 'application/json' },
  },
  // Enable connection pooling

    autoRefreshToken: true,
    persistSession: true,
  },
```

### Caching
```javascript
// Add caching for agent list
const agentCache = new Map();
const CACHE_TTL = 5 * 60 * 1000; // 5 minutes

app.get('/agents', (req, res) => {
  if (agentCache.has('agents')) {
    const { data, timestamp } = agentCache.get('agents');
    if (Date.now() - timestamp < CACHE_TTL) {
      return res.json(data);
    }
  }
  
  const agents = listAgents();
  agentCache.set('agents', { data: agents, timestamp: Date.now() });
```
