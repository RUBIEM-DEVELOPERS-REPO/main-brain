# Research: As a Technical Writer, write release notes that lead with user-facing benefit

**Release‑Notes Style Guide – Lead with the User‑Facing Benefit**

When you write release notes for NeuroWorks (the local AI‑workforce platform), start every entry by stating *what the user gains* before describing the technical change. This makes the notes instantly useful, improves adoption, and reduces support friction.

| Step | What to write | Why it works |
|------|---------------|--------------|
| **1. Benefit headline** | “*You can now run parallel AI‑agents without leaving the chat window*.” | Puts the user’s goal front‑and‑center, answering “What’s in it for me?” immediately. |
| **2. Brief description** | One‑sentence “We added a new “Team Dispatch” UI that lets you launch multiple agents from a single command.” | Gives context while keeping the focus on the benefit. |
| **3. Technical details (optional)** | Bullet list of the underlying changes (API endpoint, UI component, performance improvement). | Provides the depth that developers or power users need without obscuring the benefit. |
| **4. Impact & actions** | “No migration needed. Existing workflows continue to work.” | Clarifies any required user action and reassures stability. |

### Example Release Note (v2.4.0)

> **You can now schedule AI‑driven reports to run overnight.**  
> • Added a “Scheduled Tasks” panel that lets you define start time, recurrence, and output format.  
> • New API endpoint `/schedule` handles task queuing with priority handling.  
> • No changes required to existing bots; they will appear in the schedule list automatically.

### Tips from the Research Vault

* Keep the benefit statement **short (≤ 12 words)** and **action‑oriented**.  
* Follow the benefit with a **single‑sentence “what’s new”** line, then optional technical bullets.  
* Use the same structure for every release to create a predictable, scan‑friendly format【0-Inbox/202605290928-research-as-a-technical-writer-write-release-notes-that-lead-with-use.md】.  

By consistently leading with the user‑facing benefit, release notes become a quick‑read value proposition rather than a wall of technical jargon. This approach aligns with NeuroWorks’ goal of empowering users to get more done with less friction.
