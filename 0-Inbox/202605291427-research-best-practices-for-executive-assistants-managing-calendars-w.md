# Research: best practices for executive assistants managing calendars with deep work blocks and timezone awareness

**Executive‑assistant calendar‑management best practices for deep‑work blocks and timezone awareness**

| Goal | Practical steps | Why it works |
|------|----------------|--------------|
| **Protect deep‑work time** | 1. **Create recurring “focus” blocks** (e.g., 2‑hour windows) on the executive’s calendar and mark them *busy* with a clear label such as “🟢 Deep‑Work – No Meetings”.<br>2. **Add a buffer** (15‑30 min) before and after each block to absorb overruns and give the executive a mental transition period.<br>3. **Set default meeting length** to 30 min (or the executive’s preferred short slot) so ad‑hoc requests are less likely to encroach on focus time. | Repeating, labeled blocks make the expectation visible to the whole organization and reduce “meeting‑cannonball” requests. Buffers protect the cognitive ramp‑up/down needed for high‑quality work. |
| **Coordinate across time zones** | 1. **Store the executive’s primary time zone** in the calendar system (e.g., Outlook/Google) and enable *time‑zone display* for all attendees.<br>2. **Use “time‑zone‑aware invites”**: when scheduling a meeting, select the executive’s zone as the reference; the platform will automatically show the local time for each participant.<br>3. **Maintain a shared “world‑clock” sheet** (Google Sheet or Confluence page) listing the executive’s core working hours in each relevant zone. Link this sheet in the calendar description for quick reference.<br>4. **Apply “meeting‑window rules”**: only schedule cross‑zone calls within a defined overlap (e.g., 9 am‑12 pm PST ↔ 5 pm‑8 pm CET). Outside this window, suggest asynchronous alternatives (recorded video, shared docs). | Explicit time‑zone handling prevents accidental off‑hours invites and respects participants’ work‑life balance. A shared world‑clock makes the rule transparent and reduces back‑and‑forth clarification. |
| **Prioritise and triage requests** | 1. **Implement a “request‑gate”**: all meeting proposals first land in a shared inbox or form (e.g., Microsoft Forms). The assistant reviews, tags (Urgent/Info/Optional), and either schedules, defers, or declines.<br>2. **Leverage “meeting‑type templates”** – e.g., “Quick Sync (15 min)”, “Strategic Review (60 min)”, “Deep‑Work Block (2 h)”. Templates pre‑populate agenda fields and required prep, ensuring only meetings with clear purpose are booked.<br>3. **Set “auto‑decline” rules** for conflicts with deep‑work blocks or outside the agreed cross‑zone window. | A gate and templates enforce discipline, keep the executive’s calendar lean, and give requestors a clear path to get time when truly needed. |
| **Communicate the policy** | 1. Draft a short **calendar‑use guideline** (1‑page) and circulate to the executive’s team and key stakeholders.<br>2. Pin the guideline to the executive’s calendar description and to the shared world‑clock sheet.<br>3. Periodically (quarterly) review usage metrics (e.g., % of deep‑work blocks protected) and adjust rules as needed. | Transparency builds buy‑in; regular review ensures the system stays aligned with evolving priorities. |

### Quick‑reference checklist for the assistant

- [ ] Mark deep‑work blocks as **busy** and include the “🟢 Deep‑Work – No Meetings” label.  
- [ ] Add 15‑30 min buffers before/after each block.  
- [ ] Verify the executive’s primary time zone is set in the calendar app.  
- [ ] Use the calendar’s *time‑zone‑aware* invite feature for all meetings.  
- [ ] Check the shared world‑clock sheet for overlap windows before confirming cross‑zone calls.  
- [ ] Route new meeting requests through the request‑gate form.  
- [ ] Apply template‑based agendas and auto‑decline conflicts.  
- [ ] Update the calendar‑use guideline annually and share any changes.
