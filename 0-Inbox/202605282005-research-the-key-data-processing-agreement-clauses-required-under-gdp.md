**Key DPA clauses that GDPR requires (Article 28 § 3 and Recital 81)**  

| Required clause (what the DPA must set out) | GDPR reference |
|---|---|
| **1. Subject‑matter and duration of the processing** – what personal data will be processed, for what purpose and for how long. | Art. 28 § 3 ( “subject‑matter and duration of the processing” ) |
| **2. Nature and purpose of the processing** – description of the activities the processor will carry out. | Art. 28 § 3 ( “nature and purpose of the processing” ) |
| **3. Type of personal data and categories of data subjects** – e‑mail addresses, health data, customers, employees, etc. | Art. 28 § 3 ( “type of personal data and categories of data subjects” ) |
| **4. Obligations and rights of the controller** – controller’s instructions, responsibility for compliance, etc. | Art. 28 § 3 ( “obligations and rights of the controller” ) |
| **5. Processor‑only‑on‑written‑instructions** – the processor may not act except on the controller’s documented instructions. | Art. 28 § 3 ( “processor shall process personal data only on documented instructions …” ) |
| **6. Confidentiality** – anyone with access to the data must be bound by confidentiality. | Art. 28 § 3 ( “persons authorised to process the personal data shall be under an appropriate confidentiality obligation” ) |
| **7. Technical and organisational security measures** – measures that meet the standard of Article 32. | Art. 28 § 3 ( “appropriate technical and organisational measures … to ensure a level of security appropriate to the risk” ) |
| **8. Sub‑processing** – processor may not engage another processor without the controller’s prior written consent and must flow‑down the same DPA obligations. | Art. 28 § 4 (sub‑processor clause) |
| **9. Assistance to the controller** – help with data‑subject rights, DPIAs, and prior consultation with supervisory authorities. | Art. 28 § 3 ( “assist the controller … in complying with the obligations … concerning the rights of data subjects, security of processing (Art. 32) and prior consultation (Art. 36)” ) |
| **10. Return or deletion of data on termination** – processor must erase or return all personal data at the end of the contract. | Art. 28 § 3 ( “upon termination of the contract, the processor shall, at the choice of the controller, delete or return all personal data” ) |
| **11. Audit and inspection rights** – controller may audit the processor and obtain evidence of compliance. | Art. 28 § 3 ( “allow and contribute to audits, inspections … and provide all information necessary to demonstrate compliance” ) |

**Why these clauses matter**  
Recital 81 requires the controller to use only processors that provide “sufficient guarantees … to implement technical and organisational measures” (i.e., security and compliance guarantees) — the clauses above operationalise that requirement 【2†L31-L38】.  

Together, the clauses form the legally binding DPA that the GDPR makes mandatory whenever a controller engages a processor 【1†L13-L20】【2†L9-L15】.
