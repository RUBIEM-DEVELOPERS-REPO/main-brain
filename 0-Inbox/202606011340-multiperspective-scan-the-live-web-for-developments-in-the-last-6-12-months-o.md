The request is to identify developments over the past 6‑12 months related to “scanning” (software, apps, and general scanning practices), focusing on news, blog posts, product releases, or official announcements, and to note any changes compared with earlier information.

### critical

### practitioner

### recent  
The “recent” perspective also yields only the identical set of product pages and the generic tutorial. No recent news articles, press releases, or version‑change logs were identified that would signal a shift from earlier information [1‑4].

## Cross‑cutting themes  
- **Feature continuity:** All app listings emphasize AI‑based background removal, OCR, multi‑device sync, and PDF editing, indicating these capabilities remain central [2].  
- **Platform support:** Windows Scan is still distributed via the Microsoft Store (version 2026.5.27.269) and HP Easy Scan targets macOS only [1][4].  
- **Instructional guidance:** The Computer Hope guide continues to advise using built‑in Windows Fax and Scan or scanner‑specific software for basic scanning steps [3].  

## Open questions  
- **What new versions or feature roll‑outs have occurred in the past year?** No release notes or changelogs were found, leaving the timing of any updates ambiguous.  
- **Are there emerging security or privacy concerns with AI‑enhanced scanning apps?** The critical perspective supplies no analysis, so any recent controversies are undocumented.  
- **How have professional workflows (e.g., remote work, tax filing) adapted to new scanning tools?** While marketing copy mentions tax‑documentation workflows and offline editing, concrete adoption data or user feedback are absent.  

## Bottom line  
The available evidence does not reveal any substantive developments in scanning software or practices over the last 6‑12 months beyond the baseline feature set already advertised (AI‑enhanced OCR, multi‑device sync, PDF editing). All perspectives converge on the same product descriptions and generic how‑to guidance, and no distinct announcements, version changes, or critical analyses were identified. Consequently, any claim of “new” developments cannot be substantiated from the current sources; further targeted searching (e.g., press‑release archives, developer blogs) would be required to uncover recent changes.
