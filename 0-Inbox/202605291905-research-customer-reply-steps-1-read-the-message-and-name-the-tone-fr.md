# Research: customer reply. Steps: 1) Read the message and name the tone (frustrated / confused / excited / neutral). 2) Identify the underlying need (often different from the literal ask). 3) Draft a reply that opens with the right acknowledgment, resolves the real need, attaches a date to any commitment. 4) Flag any churn-risk or expansion-signal language separately. No macro-speak

**1. Tone** – *Neutral* (the request is procedural, no emotion is expressed).  

**2. Underlying need** – The customer (or internal stakeholder) wants a ready‑to‑send reply that:  
- acknowledges the received message,  
- solves the real problem hidden behind the literal ask,  
- includes a concrete date for any promised action, and  
- highlights any churn‑risk or upsell signal for the CS team.  

**3. Draft reply**

> Hi [Name],  
>   
> Thanks for reaching out – I understand you’re looking for [brief restatement of the real need, e.g., “a faster way to pull reports from NeuroWorks”]. I’ve checked the details and can have a solution ready for you by **Monday, June 10 (2026)**.  
>   
> Here’s what I’ll do:  
> 1. [Step 1 – e.g., “Create a custom query in the vault that extracts the required fields.”]  
> 2. [Step 2 – e.g., “Set up a scheduled task so the report runs automatically each week.”]  
>   
> I’ll send you the final configuration and a short walkthrough video on **June 10**. If anything changes before then, I’ll let you know right away.  
>   
> Please let me know if this timeline works for you or if there’s anything else you’d like to add.  
>   
> Best,  
> [Your Name]  
> Customer Success, NeuroWorks  

**4. Churn‑risk / expansion signal**  
- *Signal*: The request for a “faster way to pull reports” suggests the user is hitting performance limits – an opportunity to propose a premium analytics add‑on or dedicated compute resources.  
- *Action*: Flag this ticket for the Account‑Growth team to discuss possible upsell during the next check‑in.
