# Platform portability — options A/B/C/D

A. Local-first, zero-key desktop app (lowest lift, ships fastest)One installer (Tauri/Electron wrapper, or "Docker Desktop one-click"). First run = a setup wizard, not a .env. Default path is fully local, zero keys: Ollama bundled, executor pinned to clawbot (not Hermes), vault = a local folder the user picks, GitHub sync off by default. Cloud providers (OpenRouter, Mailjet, Stripe) become optional fields in an in-app Settings page that writes to a per-install encrypted config store. Each person runs their own instance.  You ship a product nobody needs your keys for; everyone is single-tenant on their own box.B. Self-host-in-a-box (docker compose up) (middle lift, no ops burden for you)Ship docker-compose.yml bundling server + web + Ollama + a storage volume. docker compose up  working instance, zero external keys, fully local LLM. Already half-built (you have a Dockerfile/compose from the storage work). Pro/cloud features are opt-in via the Settings UI. Orgs self-host; you carry no servers, no per-user data, no billing liability.  The "open-source product" path.C. Hosted multi-tenant SaaS (highest lift, highest ceiling)You run one cloud deployment; users sign up and get an isolated workspace. Your platform keys (OpenRouter/Mailjet/Stripe) are shared infrastructure — users never see a key; they pay a subscription (Stripe is already wired). Requires real multi-tenancy: per-tenant data isolation, vault moved off a shared GitHub repo to per-tenant object storage, auth hardening, usage metering, and dropping the seeded-admin pattern.  The business, but months of work and you own everyone's data + uptime.D. Hybrid: free local core + managed connector relay (phased; B then a slice of C)Core = Approach B (free, self-host, zero keys). The genuinely hard-to-BYO pieces — email send and payments — are offered through a managed relay you run: users OAuth into the relay instead of pasting Mailjet/Stripe keys, metered and billed. Everything else stays local.  Keeps the free, key-free promise while monetizing only the parts users can't easily self-provision.
