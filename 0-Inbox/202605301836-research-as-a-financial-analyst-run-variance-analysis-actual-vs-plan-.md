# Research: As a Financial Analyst, run variance analysis: actual vs plan vs prior period, with explanations

**Variance analysis – actual vs. plan vs. prior period (with explanations)**  

| Item | Actual (period N) | Plan (budget N) | Prior period (N‑1) | Variance % vs Plan | Variance % vs Prior | Likely explanation* |
|------|------------------|----------------|--------------------|-------------------|--------------------|----------------------|

\*Explanations are typical drivers; replace with the specific factors you uncover in your data (e.g., product‑level performance, market‑trend impact, pricing actions, cost‑saving initiatives, one‑off items).

### How to run the analysis in practice  

1. **Gather the three data sets** – actual results for the current period, the approved budget/plan for the same period, and the actual results from the prior comparable period (e.g., same month last year).  
2. **Normalize the figures** – ensure all amounts are in the same currency and accounting basis (e.g., GAAP, IFRS).  
3. **Calculate absolute and percentage variances**:  
   - *Variance vs. Plan* = Actual − Plan  
   - *% Variance vs. Plan* = (Actual − Plan) / Plan × 100%  
   - *Variance vs. Prior* = Actual − Prior  
   - *% Variance vs. Prior* = (Actual − Prior) / Prior × 100%  
4. **Create a variance waterfall** – stack the line‑item variances to see which components drive overall performance.  
5. **Add narrative explanations** – for each significant variance (typically > 5 % or material dollar amount), investigate the underlying cause (sales mix, pricing, volume, cost changes, one‑off events). Document the driver and any corrective actions.  
6. **Present the results** – a concise table (as above) plus a short commentary slide that highlights the key take‑aways for senior management.
