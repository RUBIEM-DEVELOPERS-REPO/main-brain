# Research: status report format with sections Done In progress Blocked Next

A “status report” is simply a short update that tells stakeholders what the current **status** (i.e., condition or state) of a set of tasks is — the word *status* itself means “stav” or “situace” in Czech, i.e., the present situation of something [1].  

Because the term does not prescribe a particular layout, teams often adopt a very simple, column‑ or section‑based template that groups work items by their current condition. The most common sections are:

| Section | What to list |
|---------|--------------|
| **Done** | Tasks that have been completed since the last report. |
| **In progress** | Tasks that are actively being worked on. |
| **Blocked** | Tasks that cannot move forward because of a dependency, risk, or missing resource. |
| **Next** | Tasks that will be started after the current work finishes (the upcoming backlog). |

Each entry typically includes a brief title, an owner, and optionally a short note (e.g., % complete, expected finish date, reason for blockage). This structure lets readers quickly see what has been finished, what is ongoing, what is stuck, and what will come next—essentially a snapshot of the project’s **status** as defined by the term itself [1].

*Note:* The supplied source only defines the word “status” and does not describe a specific report format, so the above template is a widely‑used convention rather than something directly documented in the evidence.
