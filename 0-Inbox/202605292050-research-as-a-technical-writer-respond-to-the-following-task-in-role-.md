### 1. Purpose  
Create a short, clear description of the “Ask Tao anything” free‑form task that can be used in documentation, onboarding material, or a help‑center article.

### 2. Audience  
- **Primary:** End‑users who will interact with the “Ask Tao anything” feature.  
- **Secondary:** Support staff and product managers who need a concise reference.

### 3. Key Points to Communicate  

| Element | What to Say | Reason |
|---------|-------------|--------|
| **Feature name** | *Ask Tao anything* | Identifies the function. |
| **What it does** | Allows users to pose any question to Tao, the system’s AI assistant, and receive an immediate, context‑aware response. | Sets expectation of open‑ended interaction. |
| **Interaction style** | Free‑form, natural‑language input; no predefined templates or categories. | Highlights flexibility. |
| **Role of the Technical Writer** | Drafts the user‑facing description, ensures terminology is consistent, and provides examples that illustrate typical queries. | Clarifies the writer’s responsibility. |
| **Tone** | Friendly yet precise; avoid overly technical jargon unless the user’s query demands it. | Aligns with best‑practice guidance that “technical language… can be almost impossible to understand” when overused【1†L13-L16】. |
| **Example prompt** | *“How do I export my project data to CSV?”* | Shows a concrete use case. |
| **Typical response time** | Near‑real‑time (seconds). | Sets realistic performance expectations. |
| **Limitations** | Answers are generated based on current knowledge; complex or ambiguous queries may require clarification. | Manages user expectations. |

### 4. Suggested Documentation Snippet  

> **Ask Tao anything**  
>  
> *What it is:* A free‑form, natural‑language interface that lets you ask Tao any question—technical or non‑technical—and receive an instant, context‑aware answer.  
>  
> *How to use:* Type your query in the input box and press **Enter**. Tao will respond within seconds.  
>  
> *When to use:* Ideal for quick look‑ups, troubleshooting steps, or exploring features without navigating multiple help articles.  
>  
> *Tips for clear answers:*  
> - Keep questions concise.  
> - Use specific terms rather than vague language.  
> - If the answer seems incomplete, re‑phrase or add details.  

### 5. Style Checklist (Technical Writing Best Practices)  

- **Clarity:** Use plain language; avoid unnecessary technical terms that could make the text “almost impossible to understand”【1†L13-L16】.  
- **Brevity:** Keep sentences under 20 words where possible.  
- **Consistency:** Capitalise the feature name (“Ask Tao anything”) every time it appears.  
- **Active voice:** “Type your query” rather than “Your query should be typed”.  
- **Scannability:** Use bullet points and bold headings for quick reference.
