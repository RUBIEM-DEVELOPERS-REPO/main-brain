# Research: As a Financial Analyst, build forecasts and models with assumptions stated upfront

**Financial‑analysis forecasting framework – assumptions first**

| Step | What to do | Why it matters (assumption focus) |
|------|------------|-----------------------------------|
| 1️⃣ Define the business question | Clearly state the decision the forecast will support (e.g., “What will net revenue be in FY 2027 if we launch Product X?”). | Sets the scope and forces you to articulate the *key drivers* you’ll assume values for. |
| 2️⃣ List all quantitative inputs | Identify revenue streams, cost categories, growth rates, pricing, volume, churn, tax rates, capital expenditures, etc. | Each input becomes an *assumption* that will be documented up front. |
| 3️⃣ Gather historical data | Pull at least 3‑5 years of actuals (or the longest reliable series) for each input. | Historical trends are the baseline from which you’ll justify forward‑looking assumptions. |
| 4️⃣ Build the base‑case model | Use a spreadsheet or modelling tool to link inputs to outputs (e.g., revenue = price × units; EBITDA = revenue – COGS – SG&A). | The base case reflects “most likely” assumptions; all formulas should be transparent. |
| 5️⃣ Document every assumption | Create a separate “Assumptions” tab (or section) that lists: <br>• Variable name <br>• Value used in the base case <br>• Source (historical trend, market research, management guidance) <br>• Reasoning / confidence level | This satisfies the “stated upfront” requirement and makes the model auditable. |
| 6️⃣ Develop scenario variants | Adjust the key assumptions to produce: <br>• **Optimistic** (e.g., higher growth, lower cost) <br>• **Pessimistic** (e.g., slower adoption, higher expense) <br>• **Stress** (e.g., regulatory shock) | Scenarios illustrate the sensitivity of outcomes to the assumptions you’ve listed. |
| 7️⃣ Run sensitivity analysis | Use data‑tables or tornado charts to show how changes in each assumption (±5‑10 %) affect the primary metric (e.g., EPS, cash flow). | Highlights which assumptions are most material and where to focus risk‑management. |
| 8️⃣ Validate & iterate | Compare model outputs with any interim actuals, get feedback from stakeholders, and revise assumptions as needed. | Keeps the forecast credible over time. |
| 9️⃣ Communicate results | Prepare a concise deck: <br>• Executive summary of forecasted outcomes <br>• Assumptions table (front‑page) <br>• Scenario outcomes <br>• Sensitivity highlights | Decision‑makers see the numbers **and** the underlying premises immediately. |
| 🔟 Archive for future use | Save the model, assumptions, and source documents in a version‑controlled repository (e.g., the NeuroWorks vault). | Enables quick updates and audit trails for later periods. |

### Key take‑aways
- **Assumptions are the foundation** – list them before any calculation.
- **Transparency**: every number in the model must trace back to a documented assumption.
- **Scenario planning**: varying assumptions shows the range of possible outcomes and helps risk assessment.
- **Iterative validation** keeps the forecast aligned with reality.
